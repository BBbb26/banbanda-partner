# 校园搭子小程序 (Campus Connect Mini Program)

## 🚀 项目简介 (Project Introduction)

“校园搭子”是一款专为在校大学生设计的社交小程序，旨在帮助同学们快速、精准地找到志同道合的伙伴，共同进行学习、运动、美食、娱乐等各类校园活动。项目致力于打破校园内的社交壁垒，让每个学生都能轻松发起和参与活动，丰富课余生活。

## ✨ 项目背景 (Background)

在现代大学生活中，尽管社交网络发达，但许多学生仍感到孤独，或在寻找有共同兴趣的伙伴参与线下活动时遇到困难。“想去打球却凑不齐人”、“想找人一起泡图书馆却不认识学霸”、“想拼单喝奶茶却找不到饭搭子”——这些都是校园生活的常见痛点。本项目应运而生，旨在通过一个轻量、便捷的平台，解决这些“找搭子”的难题。

## 🌟 主要功能 (Features)

- **智能匹配与发现**:
  - 首页采用**卡片式瀑布流**，无限下拉，高效发现潜在的“搭子”。
  - 支持按“推荐”、“最新”、“热门”等多种维度对用户卡片进行排序。
  - 集成**学校选择**和**关键词搜索**功能，实现精准查找。

- **活动广场**:
  - 用户可以发布或浏览**组队活动**，如“周末篮球局”、“图书馆学习小组”等。
  - 活动卡片清晰展示活动状态、时间、地点和已参与人数。

- **校园圈动态**:
  - 提供一个类似朋友圈的**信息流**，供用户分享图文动态，营造社区氛围。
  - 支持点赞、评论等基本社交互动。

- **即时通讯**:
  - 内置聊天功能，当用户匹配成功或对彼此感兴趣时，可**一键发起对话**，方便深入沟通。
  - 聊天功能采用**分包加载**，不影响主包性能。

- **丰富的个人中心**:
  - 用户可以编辑个人资料，通过选择**兴趣标签**来定义自己。
  - 包含“我的收藏”、“积分商城”等模块。
  - 创新的**“宠物喂养”**游戏化功能，提升用户粘性和趣味性。

## 🛠️ 技术栈 (Tech Stack)

- **前端**: 原生微信小程序 (WXML, WXSS, JavaScript ES6+)
- **核心框架**: 无，采用小程序原生开发范式。
- **状态管理**: 小程序原生 `Page.data` + `App.globalData` 模式。
- **UI 库**: 无，所有组件均为自定义实现，以保证设计风格的独特性和统一性。
- **代码风格**: 遵循 `ESLint` 规范。

## 🎨 设计亮点 (Design Highlights)

1.  **可解释性推荐算法**:
    - 本项目的核心亮点之一。在展示匹配度分数的同时，系统会用一句自然语言（如“你们都喜欢晚上去图书馆学习”）来解释推荐原因，让匹配不再是冷冰冰的数字，极大地提升了推荐的透明度和用户的信任感。

2.  **高信息密度的卡片设计**:
    - 首页的用户卡片在有限空间内，通过精巧的布局融合了封面、匹配度、标签、信誉分、社交证明（Social Proof）等多种元素，帮助用户在快速浏览中高效获取关键信息，做出判断。

3.  **手绘与 Emoji 风格的 UI**:
    - 整个小程序采用了一种轻松、活泼的手绘风格，并大量使用 Emoji 作为图标和装饰，营造出年轻、友好的社区氛围，精准命中目标用户群体的审美。

4.  **游戏化体验设计**:
    - “宠物喂养”功能是提升用户留存的巧妙设计。它将用户的活跃行为（如登录、发布动态）转化为游戏内的激励，增加了产品的趣味性和长期吸引力。

## 🤖 AI 协作流程 (AI Collaboration Flow)

本项目在开发过程中，与 Gemini 等大语言模型进行了深度协作，覆盖了从代码分析到文档撰写的多个环节，显著提升了开发效率和代码质量。

1.  **代码分析与理解**: AI 辅助分析了整个项目的代码结构，快速梳理出用户交互流程、页面结构、数据流和状态管理模式。
2.  **项目亮点挖掘**: AI 从代码的数据结构中，成功推断并总结了“可解释性推荐算法”这一核心产品亮点。
3.  **专业文档撰写**: 本 `README.md` 文件以及项目的深度分析报告，均由 AI 辅助生成。AI 能够根据要求，产出结构清晰、语言专业、可直接用于作品集或面试的文档。
4.  **技术方案建议**: AI 针对项目现状，提出了包括“组件化封装”、“长列表性能优化”、“全局状态管理演进”等多项具有建设性的技术优化建议。
